**Standing lessons (carry forward):**
- **pass@k (k≥4, n≥8) is the metric** — greedy pass@1 (±2/25 noise) and the SDPO loss are both blind to regression.
- **Select training data by measured solvability** (learnability frontier), not difficulty label.
- SDPO on the frontier = **capability elicitation/sharpening** (pass@k→pass@1), bounded by base pass@k; new capability (hard) needs **live judge feedback** (iteration 2).
- **Mode collapse here = a self-distillation diversity loss** (pass@k erodes; "length-held ≠ diversity-held" — pass@k is the metric, not length): flat reward groups → weak policy gradient → self-distillation sharpens. **Lower/warmed LR only *attenuates*** (iter-06: −0.33→−0.167). **The *cure* is reviving the policy gradient via non-flat groups** — **iter-07 confirmed: frontier-band data → `flat_group` 0.75→0.44 → pass@8 regression REVERSED (+0.083), first iter to beat base.** (KL-to-base is *not* an available lever — `--beta` is inert in SDPOTrainer, `sdpo-beta-inert`.)
- **The 8B thinking-ON SDPO regime needs Modal H200 (≥141GB) — the GB10's 128GB unified memory OOM-cascades on the training backward** (full-vocab logits over ~20k-token sequences); the 512-cap smoke does NOT exercise that peak. GB10 stays the box for cheap diagnostics/eval/plotting.
- **Instrument runs to be reviewable** ([OBSERVABILITY.md](design/OBSERVABILITY.md)): persist rollout text + per-token logprobs + rich split metrics, or you can't diagnose a collapse after the fact (iter-05 saved none).
- **Spend tracking:** cross-iteration cost segmentation (Train / Eval / Infra / **Not productive**) lives in [`reports/comparison/SPEND.md`](../reports/comparison/SPEND.md) — **~35% of all Modal spend was wasted compute**, almost all of it the pre-hardening failure modes (iter-03 integration false-starts, iter-05 eval crash/retry). Classify from each iteration's PROVENANCE/REPORT, not billing (uniform app descriptions).
